Regra de Calculo de Metas
📅 Meta Diaria = Meta Mensal ÷ dias reais do mes (com 2 casas decimais — ex: R$ 6.129,03 × 31 = R$ 190.000,00 ✅)
📆 Meta Semanal = Meta Mensal ÷ 4,33 semanas (com 2 casas decimais)
📊 Super Meta = 120% da meta mensal (ou valor customizado)
👩 Meta por Vendedora = Meta Mensal ÷ nº de vendedoras ativas (com 2 casas decimais)
✅ Distribuição precisa: Ex. R$ 190.000 em 31 dias → meta diária = R$ 6.129,03 (exata, sem arredondamento que causava desvio).
A meta progressiva do dia recalcula automaticamente usando o saldo restante ÷ dias restantes do mês.

create table if not exists lancamentos (
  id bigint generated always as identity primary key,
  loja_id text references lojas(id),
  data_venda date not null,
  faturamento numeric(12,2) default 0,
  num_transacoes int default 0,
  tm_itens_venda numeric(5,2) default 0,
  clientes_total int default 0,
  clientes_novos int default 0,
  visitantes int default 0,
  observacao text,
  lancado_por text,
  atualizado_em timestamptz default now(),
  unique(loja_id, data_venda)
);

create table if not exists vendedoras (
  id text primary key,
  nome text not null,
  loja_id text references lojas(id),
  cargo text default 'ESP',
  meta_mensal numeric(12,2) default 0,
  super_meta_mensal numeric(12,2) default 0,
  ativa boolean default true,
  atualizado_em timestamptz default now()
);

create table if not exists vendas_vendedora (
  id bigint generated always as identity primary key,
  vendedora_id text references vendedoras(id),
  loja_id text references lojas(id),
  data_venda date not null,
  valor_vendido numeric(12,2) default 0,
  num_transacoes int default 0,
  lancado_por text,
  atualizado_em timestamptz default now(),
  unique(vendedora_id, data_venda)
);

create table if not exists metas_loja (
  id bigint generated always as identity primary key,
  loja_id text references lojas(id),
  mes_ref text not null,
  meta_mensal numeric(12,2) default 0,
  super_meta_mensal numeric(12,2) default 0,
  meta_semanal numeric(12,2) default 0,
  meta_diaria numeric(12,2) default 0,
  super_semanal numeric(12,2) default 0,
  super_diaria numeric(12,2) default 0,
  dias_mes int default 31,
  ticket_alvo numeric(8,2) default 150,
  atualizado_em timestamptz default now(),
  unique(loja_id, mes_ref)
);

🔥 PARTE 1 — Criar banco de dados no Firebase (5 minutos)
1
Acesse console.firebase.google.com com sua conta Google → clique Adicionar projeto → nome: avatim-gestao → desativar Google Analytics → Criar projeto
2
No menu lateral: Firestore Database → Criar banco de dados → Modo produção → Região: southamerica-east1 (São Paulo) → Ativar
3
No Firestore: clique Regras → substitua tudo pelo código abaixo → Publicar: 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
4
Volte à página inicial do projeto → clique </> Web → apelido: avatim-web → Registrar app → Copie os valores do firebaseConfig (apiKey, projectId, etc.)
5
Vá em Authentication → Começar → aba Sign-in method → ativar E-mail/senha. Depois em Users → adicionar um usuário por loja (ex: [email protected])
6
Abra a aba 🔥 Firebase (menu acima) → cole API Key e Project ID → clique Salvar e Testar. Se aparecer ✅ o banco está funcionando!

Renomeie o arquivo HTML para index.html → no GitHub clique uploading an existing file → arraste o index.html → Commit changes

Settings (engrenagem) → Pages → em Source: selecione Deploy from a branch → Branch: main → Save → aguarde ~2 minutos
